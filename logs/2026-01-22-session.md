# Session Log: 2026-01-22

## Goal

Implement Part 2 of the multi-brand content workflow: Blog → Social repurposing.

1. Add `/generate-social` command for transforming blog content into social posts
2. Create brand-specific `social-guidelines.md` context files
3. Support multiple platforms (LinkedIn, Twitter) with variant generation
4. Test with both brands and merge to main

**Workflow**: Blog mockup → Key message extraction → Platform-specific post generation → Brand voice application

---

## Decisions Made

### Architecture
- Reuse existing `social-content` skill for platform specs (character limits, hook formulas, templates)
- New `social-guidelines.md` files contain only brand-specific adaptations (not duplicating generic platform rules)
- Output directory: `drafts/{brand}/social/` (separate from blog mockups)

### Command Parameters
- `--brand` (required): Brand name
- `--source` (optional): Specific blog/mockup path (default: most recent mockup)
- `--platforms` (optional): Comma-separated list (default: `linkedin,twitter`)
- `--variants` (optional): Number of variants per platform (default: 2)

### Output Format
- Single file per source containing all platform variants
- YAML frontmatter with metadata
- Key messages extracted section
- Platform sections with variants, character counts, hook types
- Brand voice checklist and usage recommendations
- Naming: `{source-slug}-social-{YYYY-MM-DD}.md`

### Variant Differentiation
- Variant 1: Primary insight + story/experience hook
- Variant 2: Secondary insight + contrarian/framework hook
- Each variant uses different hook type for A/B testing
